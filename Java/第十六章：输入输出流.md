## 第十六章：输入输出流

### ① 核心概念

**一句话总结：**
> 流（Stream）是数据的有序序列，Java程序通过“流”从数据源读取数据或向目的地写入数据，就像水管中流动的水。

**生活类比：**
- **输入流** = 从水厂到家里的**进水管**（数据流入程序）
- **输出流** = 从家里排出去的**出水管**（数据流出程序）

**用C的视角理解：**
相当于C语言的`FILE*`和`fread`/`fwrite`，但Java的流体系更丰富、更面向对象。

---

### ② 流的分类

```
字节流（处理二进制数据，如图片、视频）
├── InputStream（输入字节流）
│   ├── FileInputStream（文件输入）
│   ├── BufferedInputStream（缓冲输入）
│   └── ...
└── OutputStream（输出字节流）
    ├── FileOutputStream（文件输出）
    ├── BufferedOutputStream（缓冲输出）
    └── ...

字符流（处理文本数据，如txt文件）
├── Reader（输入字符流）
│   ├── FileReader（文件输入）
│   ├── BufferedReader（缓冲输入）
│   └── ...
└── Writer（输出字符流）
    ├── FileWriter（文件输出）
    ├── BufferedWriter（缓冲输出）
    └── ...
```

**一句话区分：**
- **字节流**：万能，处理任何文件（图片、视频、文本）
- **字符流**：只处理文本，支持中文等字符编码

---

### ③ 为什么需要不同的流？

**解决什么痛点？**

1. **字节 vs 字符**：一个中文字符在GBK编码占2字节，UTF-8占3字节，字符流自动处理编码
2. **缓冲**：每次读写一个字节效率低，缓冲流批量读写提高性能
3. **装饰器模式**：可以层层包装，组合不同功能

```java
// 无缓冲：每次读一个字节，1000次磁盘操作
FileInputStream fis = new FileInputStream("file.txt");
int data;
while ((data = fis.read()) != -1) {  // 每次读一个字节
    // 处理
}

// 有缓冲：一次读8KB，只需约125次磁盘操作
BufferedInputStream bis = new BufferedInputStream(
    new FileInputStream("file.txt")
);
// 自动批量读取
```

### ④ 精简代码示例

#### 文件读取（字节流）

```java
import java.io.*;

// 读取文件内容到控制台
public class ReadFile {
    public static void main(String[] args) throws IOException {
        // 1. 创建输入流
        FileInputStream fis = new FileInputStream("test.txt");
        
        // 2. 读取数据
        int data;
        while ((data = fis.read()) != -1) {  // -1表示文件结束
            System.out.print((char) data);
        }
        
        // 3. 关闭流（释放资源）
        fis.close();
    }
}
```

#### 文件写入（字节流）

```java
import java.io.*;

// 写入数据到文件
public class WriteFile {
    public static void main(String[] args) throws IOException {
        // true表示追加写入，false或不写表示覆盖
        FileOutputStream fos = new FileOutputStream("output.txt", true);
        
        String content = "Hello Java\n";
        fos.write(content.getBytes());  // 字符串转字节数组
        
        fos.close();
    }
}
```

### ⑤ 字符流（处理文本更简单）

```java
import java.io.*;

public class CharStreamDemo {
    public static void main(String[] args) throws IOException {
        // 字符输入流：读取文本文件
        FileReader fr = new FileReader("test.txt");
        int ch;
        while ((ch = fr.read()) != -1) {
            System.out.print((char) ch);
        }
        fr.close();
        
        // 字符输出流：写入文本文件
        FileWriter fw = new FileWriter("output.txt");
        fw.write("你好，Java！");
        fw.close();
    }
}
```

### ⑥ 缓冲流（提高性能）

```java
import java.io.*;

public class BufferedStreamDemo {
    public static void main(String[] args) throws IOException {
        // 缓冲输入：一次读一行
        BufferedReader br = new BufferedReader(new FileReader("test.txt"));
        String line;
        while ((line = br.readLine()) != null) {  // 读一行
            System.out.println(line);
        }
        br.close();
        
        // 缓冲输出：高效写入
        BufferedWriter bw = new BufferedWriter(new FileWriter("output.txt"));
        bw.write("第一行");
        bw.newLine();  // 换行
        bw.write("第二行");
        bw.close();
    }
}
```

**为什么用缓冲流？**
- 无缓冲：每次写入都直接操作磁盘（慢）
- 有缓冲：先写入内存缓冲区，满了再一次性写入磁盘（快）

### ⑦ try-with-resources（自动关闭）

**一句话总结：**
> Java 7引入的语法，自动关闭实现了`AutoCloseable`接口的资源，不用手动写finally。

```java
// 传统写法（需要手动关闭）
BufferedReader br = null;
try {
    br = new BufferedReader(new FileReader("test.txt"));
    String line = br.readLine();
} catch (IOException e) {
    e.printStackTrace();
} finally {
    if (br != null) {
        try {
            br.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

// try-with-resources写法（自动关闭）
try (BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
    String line = br.readLine();
    System.out.println(line);
} catch (IOException e) {
    e.printStackTrace();
}
// br自动关闭，不需要finally
```

### ⑧ 对象序列化

**核心概念：**
> 将Java对象转换成字节序列（序列化），可以保存到文件或通过网络传输；反序列化则是将字节序列恢复成Java对象。

**生活类比：**
- **序列化** = 把乐高模型拆成零件装进盒子（对象 → 字节）
- **反序列化** = 从盒子里拿出零件重新拼成模型（字节 → 对象）

```java
import java.io.*;

// 类必须实现Serializable接口（标记接口，没有方法）
class Student implements Serializable {
    private static final long serialVersionUID = 1L;  // 版本号
    
    String name;
    int age;
    transient String password;  // transient表示不序列化
    
    Student(String name, int age, String password) {
        this.name = name;
        this.age = age;
        this.password = password;
    }
    
    public String toString() {
        return name + "," + age + "," + password;
    }
}

public class SerializeDemo {
    public static void main(String[] args) throws Exception {
        // 序列化：对象 → 文件
        Student s1 = new Student("小明", 20, "123456");
        ObjectOutputStream oos = new ObjectOutputStream(
            new FileOutputStream("student.dat")
        );
        oos.writeObject(s1);
        oos.close();
        
        // 反序列化：文件 → 对象
        ObjectInputStream ois = new ObjectInputStream(
            new FileInputStream("student.dat")
        );
        Student s2 = (Student) ois.readObject();
        ois.close();
        
        System.out.println(s2);  // 小明,20,null（password是null）
    }
}
```

**关键点：**
- `Serializable`：标记接口，告诉JVM这个类可以序列化
- `transient`：修饰的字段不会被序列化（如密码、敏感信息）
- `serialVersionUID`：版本号，反序列化时校验版本一致性

### ⑨ File类（文件和目录操作）

```java
import java.io.File;

public class FileDemo {
    public static void main(String[] args) {
        File file = new File("test.txt");
        
        // 常用方法
        file.exists();        // 是否存在
        file.isFile();        // 是否是文件
        file.isDirectory();   // 是否是目录
        file.length();        // 文件大小（字节）
        file.getName();       // 文件名
        file.getPath();       // 路径
        file.getAbsolutePath(); // 绝对路径
        file.delete();        // 删除
        
        // 创建目录
        File dir = new File("mydir");
        dir.mkdir();          // 创建单级目录
        dir.mkdirs();         // 创建多级目录
        
        // 列出目录内容
        File[] files = dir.listFiles();
        for (File f : files) {
            System.out.println(f.getName());
        }
    }
}
```
