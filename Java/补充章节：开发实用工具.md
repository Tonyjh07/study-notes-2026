# 第一部分：版本控制工具（Git）

### ① 核心概念

**一句话总结：**
> Git是分布式版本控制系统，记录文件的每一次修改，让你可以随时回退到任意历史版本，多人协作时自动合并代码。

**生活类比：**
- **无版本控制** = 写论文时手动保存 `论文_v1.doc`、`论文_v2.doc`、`论文_最终版.doc`、`论文_最终版2.doc`（混乱）
- **有Git** = 像游戏存档一样，每次修改都有记录，随时读档，多人可同时编辑不同章节

---

### ② 为什么需要Git？

**解决什么痛点？**

```java
// 痛点1：代码改坏了想回退
// 修改前：程序正常运行
// 修改后：程序崩溃，但已经记不清改了哪里

// 痛点2：多人协作冲突
// 你改了文件A，同事也改了文件A，谁的覆盖谁的？

// 痛点3：实验性功能
// 想尝试新思路，但又怕影响稳定代码
```

**Git的解决方案：**
1. **版本历史**：每次提交都是一次"存档"，随时回退
2. **分支管理**：不同功能在不同分支开发，互不干扰
3. **合并机制**：自动合并，冲突时提示人工解决

---

### ③ Git核心概念

```
工作区 (Working Directory)    暂存区 (Staging Area)    本地仓库 (Local Repo)
    │                              │                         │
    ├─ 你正在编辑的文件             ├─ 准备提交的文件          ├─ 已提交的历史版本
    │                              │                         │
    │      git add .               │      git commit         │
    └─────────────────────────────→└────────────────────────→┘
    
                                                         │
                                                         │ git push / git pull
                                                         ↓
                                                  远程仓库 (Remote Repo)
                                                  （GitHub/Gitee/GitLab）
```

**一句话理解：**
> 工作区是你正在改的，暂存区是准备拍照的，本地仓库是相册，远程仓库是云备份。

### ④ 常用Git命令

#### 初始配置

```bash
# 配置用户名和邮箱（第一次使用）
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"

# 查看配置
git config --list
```

#### 基本操作

```bash
# 1. 初始化仓库（在项目目录下）
cd my-project
git init                    # 创建.git隐藏目录

# 2. 查看状态
git status                  # 查看哪些文件被修改了

# 3. 添加文件到暂存区
git add Hello.java          # 添加单个文件
git add .                   # 添加所有修改的文件

# 4. 提交到本地仓库
git commit -m "首次提交：Hello World程序"

# 5. 查看提交历史
git log                     # 查看所有提交记录
git log --oneline           # 简洁版一行一个提交

# 6. 回退版本
git reset --hard HEAD^      # 回退到上一个版本
git reset --hard 7a8b9c     # 回退到指定commit id

# 7. 查看修改差异
git diff                    # 查看工作区与暂存区的差异
git diff HEAD               # 查看工作区与最新提交的差异
```

#### 分支操作

```bash
# 分支类比：树干是主分支，树枝是功能分支

# 1. 查看分支
git branch                  # 查看本地分支（*表示当前分支）

# 2. 创建分支
git branch feature-login    # 创建login功能分支

# 3. 切换分支
git checkout feature-login  # 切换到login分支
git switch feature-login    # 新写法（推荐）

# 4. 创建并切换（合并写法）
git checkout -b feature-login

# 5. 合并分支（先切回主分支）
git checkout main
git merge feature-login     # 把login分支合并到main

# 6. 删除分支
git branch -d feature-login
```

#### 远程仓库（GitHub）

```bash
# 1. 克隆远程仓库
git clone https://github.com/你的用户名/项目名.git

# 2. 查看远程仓库
git remote -v

# 3. 添加远程仓库
git remote add origin https://github.com/用户名/项目名.git

# 4. 推送到远程
git push -u origin main     # -u：第一次推送，建立关联

# 5. 拉取远程更新
git pull origin main        # 下载并合并
git fetch origin main       # 只下载，不合并
```

### ⑤ 典型工作流程（单人）

```bash
# 1. 开始新功能
git checkout -b feature-chat

# 2. 编写代码...
# 3. 查看修改
git status
git diff

# 4. 提交
git add .
git commit -m "完成聊天功能"

# 5. 切回主分支，合并
git checkout main
git merge feature-chat

# 6. 推送到远程
git push origin main

# 7. 删除功能分支
git branch -d feature-chat
```

### ⑥ 典型工作流程（多人协作）

```bash
# 你（开发者A）
git pull origin main        # 1. 先拉取最新代码
git checkout -b feature-aaa # 2. 创建自己的分支
# 3. 编写代码...
git add .
git commit -m "完成功能A"
git push origin feature-aaa # 4. 推送自己的分支
# 5. 在GitHub上发起Pull Request (PR)

# 同事（开发者B）
git pull origin main
git checkout -b feature-bbb
# ...类似流程

# 项目负责人
# 审核PR，合并到main分支
```

# 第二部分：构建工具（Maven/Gradle）

### ① 核心概念

**一句话总结：**
> Maven/Gradle是自动化构建工具，管理项目依赖、编译、测试、打包等全过程。

**生活类比：**
- **无构建工具** = 手工做菜（找食材、切菜、炒菜全手动）
- **有构建工具** = 智能厨房（自动配菜、自动烹饪、自动摆盘）

---

### ② 为什么需要构建工具？

**解决什么痛点？**

```xml
<!-- 痛点1：手动管理JAR包 -->
<!-- 需要找到正确的JAR、下载、复制到lib目录、还要解决版本冲突 -->

<!-- 痛点2：手动编译 -->
javac -d classes src/com/company/*.java src/com/company/model/*.java ...

<!-- 痛点3：手动打包 -->
jar cvf myapp.jar -C classes .

<!-- 痛点4：手动测试 -->
<!-- 运行所有测试类，检查结果 -->
```

**构建工具的解决方案：**
1. **依赖管理**：在配置文件中声明，自动下载JAR
2. **标准化构建**：一个命令完成编译、测试、打包
3. **生命周期**：`mvn clean install` 一键完成全流程

---

### ③ Maven入门

#### 核心概念：POM（Project Object Model）

```xml
<!-- pom.xml 项目核心配置文件 -->
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0">
    <!-- 基础信息 -->
    <groupId>com.network</groupId>      <!-- 组织/公司ID -->
    <artifactId>chat-app</artifactId>   <!-- 项目名称 -->
    <version>1.0-SNAPSHOT</version>     <!-- 版本号 -->
    <packaging>jar</packaging>          <!-- 打包类型 -->
    
    <!-- 依赖管理：自动下载JAR -->
    <dependencies>
        <!-- MySQL驱动 -->
        <dependency>
            <groupId>mysql</groupId>
            <artifactId>mysql-connector-java</artifactId>
            <version>8.0.31</version>
        </dependency>
        
        <!-- 日志框架 -->
        <dependency>
            <groupId>org.slf4j</groupId>
            <artifactId>slf4j-api</artifactId>
            <version>2.0.9</version>
        </dependency>
    </dependencies>
    
    <!-- 编译配置 -->
    <properties>
        <maven.compiler.source>11</maven.compiler.source>
        <maven.compiler.target>11</maven.compiler.target>
    </properties>
</project>
```

#### Maven标准目录结构

```
my-project/
├── pom.xml                    # Maven配置文件
├── src/
│   ├── main/
│   │   ├── java/              # Java源代码
│   │   │   └── com/network/chat/
│   │   │       └── ChatClient.java
│   │   └── resources/         # 资源文件（配置文件、图片）
│   │       └── config.properties
│   └── test/
│       └── java/              # 测试代码
│           └── com/network/chat/
│               └── ChatClientTest.java
└── target/                    # 编译输出（自动生成）
    ├── classes/               # .class文件
    └── chat-app-1.0-SNAPSHOT.jar
```

#### Maven常用命令

```bash
# 清理（删除target目录）
mvn clean

# 编译（生成target/classes）
mvn compile

# 测试（运行test目录下的测试）
mvn test

# 打包（生成jar/war）
mvn package

# 安装到本地仓库（供其他项目使用）
mvn install

# 清理+打包（最常用）
mvn clean package

# 跳过测试
mvn clean package -DskipTests
```

**Maven生命周期：**
```
clean → compile → test → package → install → deploy
```

### ④ Gradle入门

**Gradle vs Maven：**
- Maven：XML配置，规范严格，学习曲线平缓
- Gradle：Groovy/Kotlin DSL，更灵活，构建速度更快

#### Gradle配置文件（build.gradle）

```gradle
// build.gradle (Groovy DSL)
plugins {
    id 'java'
    id 'application'
}

// 项目信息
group = 'com.network'
version = '1.0-SNAPSHOT'

// 指定Java版本
java {
    sourceCompatibility = '11'
}

// 依赖管理
repositories {
    mavenCentral()           // 从Maven中央仓库下载
}

dependencies {
    // 编译依赖
    implementation 'mysql:mysql-connector-java:8.0.31'
    implementation 'org.slf4j:slf4j-api:2.0.9'
    
    // 测试依赖
    testImplementation 'org.junit.jupiter:junit-jupiter:5.9.0'
}

// 指定主类
application {
    mainClass = 'com.network.chat.ChatClient'
}

// 打包配置
jar {
    manifest {
        attributes 'Main-Class': 'com.network.chat.ChatClient'
    }
}
```

#### Gradle常用命令

```bash
# 清理
gradle clean

# 编译
gradle compileJava

# 测试
gradle test

# 打包
gradle jar

# 运行（如果配置了application插件）
gradle run

# 清理+打包
gradle clean jar
```

---

### ⑤ 依赖范围（scope）

| 范围 | 作用 | 示例 |
|------|------|------|
| compile（默认） | 编译+测试+运行时都需要 | 业务代码依赖 |
| test | 仅测试时需要 | JUnit |
| provided | 编译+测试需要，运行时容器提供 | Servlet API |
| runtime | 运行时需要，编译不需要 | JDBC驱动实现 |

```xml
<!-- Maven示例 -->
<dependency>
    <groupId>org.junit.jupiter</groupId>
    <artifactId>junit-jupiter</artifactId>
    <version>5.9.0</version>
    <scope>test</scope>
</dependency>
```

### ⑥ 如何添加依赖（找坐标）

**方式1：从Maven中央仓库搜索**
1. 访问 https://mvnrepository.com
2. 搜索需要的库（如 "mysql connector"）
3. 选择版本，复制依赖配置

**方式2：复制到pom.xml**

```xml
<!-- 搜到的内容直接复制到<dependencies>中 -->
<dependency>
    <groupId>com.google.code.gson</groupId>
    <artifactId>gson</artifactId>
    <version>2.10.1</version>
</dependency>
```
