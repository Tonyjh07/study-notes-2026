## 第十二章：字符串处理

这一章我们分三部分讲：`String`、`StringBuilder`、`StringBuffer`。

---

## 第一部分：String类

### ① 核心概念

**一句话总结：**
> String是**不可变**的字符序列，一旦创建就不能修改，任何操作都会产生新字符串。

**生活类比：**
- String = **纸质书**——你在书上画线、做笔记，实际上是在新书（副本）上操作，原书不变
- 每次修改字符串，就像重新印一本新书

**用C的视角理解：**
C语言的字符串是字符数组，可以直接修改。Java的String更像是一个**只读的常量**，不能原位修改。

---

### ② 为什么String设计成不可变？

**解决什么痛点？**

1. **安全**：字符串常量池共享，不可变防止意外修改
2. **线程安全**：多线程环境下不需要同步
3. **缓存**：hashCode可以缓存，提高HashMap性能

---

### ③ 精简代码示例

```java
public class Main {
    public static void main(String[] args) {
        // 创建方式1：字面量（推荐）
        String s1 = "Hello";
        
        // 创建方式2：new关键字
        String s2 = new String("Hello");
        
        System.out.println(s1 == s2);        // false（地址不同）
        System.out.println(s1.equals(s2));   // true（内容相同）
        
        // 字符串不可变的证明
        String str = "Java";
        System.out.println(str);           // Java
        str.toUpperCase();                 // 产生新字符串，但没接收
        System.out.println(str);           // 还是Java（没变！）
        
        String upper = str.toUpperCase();  // 接收新字符串
        System.out.println(upper);         // JAVA
    }
}
```

---

### ④ 常用方法

```java
String s = "Hello World";

// 常用方法示例
s.length();                    // 11，获取长度
s.charAt(1);                   // 'e'，获取指定位置字符
s.substring(0, 5);             // "Hello"，截取 [0,5)
s.indexOf("World");            // 6，查找子串位置
s.equals("hello world");       // false，区分大小写比较
s.equalsIgnoreCase("hello world"); // true，忽略大小写
s.toUpperCase();               // "HELLO WORLD"
s.toLowerCase();               // "hello world"
s.trim();                      // 去除首尾空格
s.replace("World", "Java");    // "Hello Java"
s.split(" ");                  // ["Hello", "World"]，分割成数组
s.contains("Hel");             // true，是否包含子串
s.startsWith("He");            // true，是否以...开头
s.isEmpty();                   // false，是否为空
```

### ⑤ 字符串比较：== vs equals()

**核心区别：**
- `==`：比较**地址**（是否同一个对象）
- `equals()`：比较**内容**（字符序列是否相同）

```java
String s1 = "Java";
String s2 = "Java";
String s3 = new String("Java");

System.out.println(s1 == s2);      // true（常量池复用）
System.out.println(s1 == s3);      // false（不同对象）
System.out.println(s1.equals(s3)); // true（内容相同）
```

**字符串常量池：**
```java
// 字面量创建：会复用常量池中的对象
String a = "Hello";
String b = "Hello";  // b和a指向同一个对象

// new创建：强制创建新对象
String c = new String("Hello");  // 新对象，不进池
String d = c.intern();           // intern()手动放入池
```

## 第二部分：StringBuilder 和 StringBuffer

### ① 核心概念

**一句话总结：**
> StringBuilder和StringBuffer是**可变的**字符串，适合频繁修改的场景，避免创建大量临时对象。

**生活类比：**
- String = 铅笔写字（写错了只能重写）
- StringBuilder = 可擦写板（写错了擦掉重写，同一块板）

---

### ② 为什么需要它们？

**解决什么痛点？**

频繁修改字符串时，String会产生大量临时对象，效率低：

```java
// 低效：循环中产生10个临时字符串对象
String result = "";
for (int i = 0; i < 10; i++) {
    result = result + i;  // 每次循环创建新String对象
}

// 高效：只用一个StringBuilder对象
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 10; i++) {
    sb.append(i);  // 在原对象上修改
}
String result = sb.toString();
```

---

### ③ 精简代码示例

```java
// StringBuilder（线程不安全，单线程用，效率高）
StringBuilder sb = new StringBuilder("Hello");
sb.append(" World");      // Hello World
sb.insert(5, ",");        // Hello, World
sb.replace(6, 11, "Java");// Hello, Java
sb.delete(5, 6);          // Hello Java
sb.reverse();             // avaJ olleH
String result = sb.toString();

// StringBuffer（线程安全，多线程用，效率稍低）
StringBuffer sbf = new StringBuffer("Hello");
sbf.append(" World");     // 用法和StringBuilder一样
```

---

### ④ 三者对比

| 特性 | String | StringBuilder | StringBuffer |
|------|--------|---------------|--------------|
| 可变性 | 不可变 | 可变 | 可变 |
| 线程安全 | 安全（不可变） | 不安全 | 安全（同步） |
| 效率 | 拼接时低 | 最高 | 中等 |
| 使用场景 | 少量修改/常量 | 单线程大量修改 | 多线程大量修改 |

**选择建议：**
```java
// 1. 字符串不变 → String
String name = "张三";

// 2. 单线程频繁拼接 → StringBuilder
StringBuilder sb = new StringBuilder();
for (int i = 0; i < 100; i++) {
    sb.append(i);
}

// 3. 多线程频繁拼接 → StringBuffer
StringBuffer sbf = new StringBuffer();
// 多个线程同时操作同一个sbf
```
