## 第十五章：泛型

### ① 核心概念

**一句话总结：**
> 泛型就是"类型参数化"——把类型像参数一样传递，让类/方法在定义时不确定类型，使用时再指定。

**生活类比：**
- **无泛型** = 纸箱（什么都往里扔，取出来时要猜是什么）
- **有泛型** = 专用收纳盒（贴上"只放书"的标签，放错编译就不通过）

**用C的视角理解：**
类似C++的模板（template），但Java的泛型是**编译时概念**，运行时会被擦除。

---

### ② 为什么需要泛型？

**解决什么痛点？**

没有泛型时（Java 5以前），集合可以放任何类型对象：

```java
// 无泛型的问题
List list = new ArrayList();
list.add("字符串");
list.add(123);        // 可以放整数
list.add(new Student()); // 可以放Student

String s = (String) list.get(0); // 要强制转换
String s2 = (String) list.get(1); // 运行时崩溃！ClassCastException
// 因为第2个元素是Integer，不能转成String
```

有泛型后，编译器帮你检查类型：

```java
// 有泛型：类型安全
List<String> list = new ArrayList<>();
list.add("字符串");
// list.add(123);  // 编译错误！不能放整数
list.add("另一个字符串");

String s = list.get(0);  // 不需要强制转换，编译器知道是String
```

**核心作用：**
1. **类型安全**：编译时检查，避免运行时ClassCastException
2. **消除强制转换**：代码更简洁
3. **代码复用**：一套代码适用于多种类型

---

### ③ 精简代码示例

```java
import java.util.ArrayList;
import java.util.List;

// 1. 泛型类
class Box<T> {           // T是类型参数（Type的缩写）
    private T content;
    
    public void set(T content) {
        this.content = content;
    }
    
    public T get() {
        return content;
    }
}

// 2. 泛型方法
class Util {
    // <T>放在返回值前，表示这是一个泛型方法
    public static <T> T getMiddle(T[] array) {
        return array[array.length / 2];
    }
}

// 3. 泛型接口
interface Pair<K, V> {   // 两个类型参数
    K getKey();
    V getValue();
}

public class Main {
    public static void main(String[] args) {
        // 使用泛型类：创建时指定具体类型
        Box<String> stringBox = new Box<>();
        stringBox.set("Hello");
        String content = stringBox.get();  // 不需要强制转换
        
        Box<Integer> intBox = new Box<>();
        intBox.set(123);
        
        // 使用泛型方法
        String[] names = {"A", "B", "C"};
        String middle = Util.getMiddle(names);  // 类型推断为String
        
        Integer[] nums = {1, 2, 3};
        Integer midNum = Util.getMiddle(nums);  // 类型推断为Integer
    }
}
```

### ⑤ 类型擦除（理解即可）

**核心概念：**
> Java的泛型只在**编译时**存在，运行时JVM会"擦除"泛型信息，把T替换成Object或边界类型。

```java
// 代码中写的
Box<String> stringBox = new Box<>();
Box<Integer> intBox = new Box<>();

// 运行时实际是（类型擦除后）
Box stringBox = new Box();
Box intBox = new Box();

// 验证：下面输出true
System.out.println(stringBox.getClass() == intBox.getClass());
```

**影响：**
- 运行时不能判断泛型类型（`instanceof Box<String>` 非法）
- 不能new泛型数组（`new T[10]` 非法）
- 静态方法中不能使用类的泛型参数

### ⑥ 通配符（?）

**核心概念：**
> `?` 表示未知类型，用于处理泛型类型的继承关系问题。

```java
import java.util.List;
import java.util.ArrayList;

public class Main {
    public static void main(String[] args) {
        List<String> strings = new ArrayList<>();
        List<Integer> integers = new ArrayList<>();
        
        // 问题：List<String> 不是 List<Object> 的子类型
        // printList(strings);  // 编译错误！类型不匹配
        
        // 解决：使用通配符
        printWildcard(strings);   // ✅
        printWildcard(integers);  // ✅
    }
    
    // 错误写法：不能接收List<String>
    // static void printList(List<Object> list) { }
    
    // 正确写法：通配符，可以接收任何类型的List
    static void printWildcard(List<?> list) {
        // 注意：不能添加元素（除了null），因为不知道具体类型
        // list.add("hello");  // 编译错误！
        list.add(null);        // 可以加null
        
        // 可以读取，返回Object类型
        Object obj = list.get(0);
        System.out.println(list.size());
    }
}
```

### ⑦ 通配符的上界和下界

**上界通配符 `<? extends T>`：**
> 表示类型是T或T的子类（作为生产者，只读不写）

**下界通配符 `<? super T>`：**
> 表示类型是T或T的父类（作为消费者，只写不读）

```java
import java.util.List;
import java.util.ArrayList;

class Animal { }
class Dog extends Animal { }
class Cat extends Animal { }

public class Main {
    public static void main(String[] args) {
        List<Dog> dogs = new ArrayList<>();
        List<Animal> animals = new ArrayList<>();
        List<Cat> cats = new ArrayList<>();
        
        // 上界：? extends Animal，可以读，不能写
        readOnly(dogs);    // ✅ Dog是Animal的子类
        readOnly(animals); // ✅ Animal是Animal本身
        
        // 下界：? super Dog，可以写，读只能读Object
        writeOnly(dogs);    // ✅ Dog是Dog本身
        writeOnly(animals); // ✅ Animal是Dog的父类
        // writeOnly(cats);  // ❌ Cat不是Dog的父类
    }
    
    // 读取场景：上界通配符
    static void readOnly(List<? extends Animal> list) {
        Animal a = list.get(0);  // 可以读，当作Animal
        // list.add(new Dog());  // 不能写，因为不知道具体类型
    }
    
    // 写入场景：下界通配符
    static void writeOnly(List<? super Dog> list) {
        list.add(new Dog());      // 可以写Dog
        // Dog d = list.get(0);   // 不能确定读出来什么类型
        Object obj = list.get(0); // 只能读成Object
    }
}
```

**记忆口诀（PECS）：**
> **Producer Extends, Consumer Super**
> - 生产者（提供数据）：用 `extends`（只能读）
> - 消费者（接收数据）：用 `super`（只能写）
