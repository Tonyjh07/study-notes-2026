## 第九章：多态

### ① 核心概念

**一句话总结：**
> 同一个方法调用，由于对象类型不同，执行不同的行为——父类引用指向子类对象，调用重写方法时表现子类的行为。

**生活类比：**
- **“播放”按钮**：点击同一个按钮
  - 音乐播放器 → 播放音乐
  - 视频播放器 → 播放视频
  - 游戏机 → 开始游戏
  
同一个操作（调用方法），不同对象有不同实现（多态）。

**用C的视角理解：**
多态就像是C语言中的**函数指针**，父类引用指向哪个子类对象，就调用哪个子类的方法——但Java在语法层面更安全、更简洁。

---

### ② 为什么需要它？

**解决什么痛点？**

没有多态时，处理不同类型的对象需要写很多条件判断：

```java
// 没有多态：需要判断类型
void makeSound(Animal a) {
    if (a instanceof Dog) {
        ((Dog)a).bark();
    } else if (a instanceof Cat) {
        ((Cat)a).meow();
    }
}
```

有了多态，代码简洁且易扩展：

```java
// 有多态：统一调用，自动执行子类方法
void makeSound(Animal a) {
    a.sound();  // 不管什么动物，都会正确叫
}

// 新增动物类型时，不需要修改这个方法！
```

**核心作用：**
1. **解耦**：调用者不需要知道具体子类类型
2. **可扩展**：新增子类不影响现有代码
3. **灵活性**：一个方法可以处理多种类型

---

### ③ 精简代码示例

```java
// 父类
class Animal {
    void sound() {
        System.out.println("动物叫");
    }
}

// 子类1
class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("汪汪");
    }
}

// 子类2
class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("喵喵");
    }
}

// 使用多态
public class Main {
    public static void main(String[] args) {
        // 多态核心：父类引用指向子类对象
        Animal a1 = new Dog();   // 向上转型（自动）
        Animal a2 = new Cat();
        
        a1.sound();   // 输出：汪汪（实际执行Dog的sound）
        a2.sound();   // 输出：喵喵（实际执行Cat的sound）
    }
}
```

### ④ 向上转型与向下转型

**一句话总结：**
> 向上转型自动安全，向下转型需要强制且可能失败。

```java
class Animal {
    void sound() { }
}

class Dog extends Animal {
    void sound() { }
    void fetch() {                      // 子类特有方法
        System.out.println("接飞盘");
    }
}

public class Main {
    public static void main(String[] args) {
        // 1. 向上转型（自动，安全）
        Animal a = new Dog();    // Dog → Animal，自动转换
        a.sound();               // ✅ 父类有这个方法
        // a.fetch();            // ❌ 编译错误！父类没有fetch()
        
        // 2. 向下转型（强制，有风险）
        Dog d = (Dog) a;         // 强制转换回Dog
        d.fetch();               // ✅ 现在可以调用了
        d.sound();               // ✅ 也能调用
        
        // 3. 不安全的向下转型
        Animal a2 = new Animal();
        // Dog d2 = (Dog) a2;    // 编译通过，但运行报错！ClassCastException
        // 因为a2实际不是Dog对象
    }
}
```

### ⑤ instanceof 运算符

**① 核心概念**
> `instanceof` 用于判断对象是否是某个类型（或其子类）的实例，避免错误的向下转型。

**② 精简代码示例**

```java
Animal a1 = new Dog();
Animal a2 = new Animal();

System.out.println(a1 instanceof Dog);    // true，a1是Dog对象
System.out.println(a1 instanceof Animal); // true，Dog是Animal的子类
System.out.println(a2 instanceof Dog);    // false，a2不是Dog

// 安全转型：先判断再转换
if (a1 instanceof Dog) {
    Dog d = (Dog) a1;    // 安全
    d.fetch();
}
```

### ⑥ 编译时类型 vs 运行时类型

**一句话总结：**
> 编译时看“左边类型”（能调用哪些方法），运行时看“右边对象”（实际执行谁的方法）。

```java
class Animal {
    void sound() { System.out.println("动物叫"); }
}

class Dog extends Animal {
    @Override
    void sound() { System.out.println("汪汪"); }
    void fetch() { System.out.println("接飞盘"); }
}

Animal a = new Dog();  // 编译时类型：Animal，运行时类型：Dog

// 编译时：编译器检查 Animal 类是否有 sound() → ✅ 有
// 运行时：实际执行 Dog 类的 sound() → 输出"汪汪"
a.sound();   

// 编译时：编译器检查 Animal 类是否有 fetch() → ❌ 没有
// a.fetch();  编译错误！

// 总结：能调什么方法看左边（编译时），怎么执行看右边（运行时）
```

**记忆口诀：**
> 编译看左边，运行看右边（方法重写）
> 属性不重写，看左边（属性没有多态）

### ⑦ 小思考题

```java
class Fruit {
    String name = "水果";
    void show() {
        System.out.println(name);
    }
}

class Apple extends Fruit {
    String name = "苹果";
    @Override
    void show() {
        System.out.println(name);
    }
}

Fruit f = new Apple();
System.out.println(f.name);    // 输出什么？
f.show();                      // 输出什么？
```

提示：属性没有多态，方法有多态。

### ⑧ 网络工程场景预告

多态在网络编程中非常常用：

```java
// 不同的网络协议，统一调用
interface Protocol {
    void handleRequest();
}

class HttpProtocol implements Protocol {
    public void handleRequest() { /* HTTP处理 */ }
}

class TcpProtocol implements Protocol {
    public void handleRequest() { /* TCP处理 */ }
}

// 服务器核心代码不关心具体协议
void process(Protocol p) {
    p.handleRequest();  // 多态！具体协议做什么，由对象决定
}
```
