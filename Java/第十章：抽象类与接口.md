## 第十章：抽象类与接口

### ① 抽象类

**① 核心概念**

**一句话总结：**
> 抽象类是不能被实例化的"半成品"类，用于定义子类的公共模板，可以包含具体方法和抽象方法。

**生活类比：**
- **抽象类** = "水果"这个概念——你不能直接"买一个水果"（无法实例化），但"苹果"和"香蕉"继承它，都有"吃"的具体方式

**用C的视角理解：**
抽象类就像是一个定义了部分方法的**结构体+函数指针**，强制子类必须实现某些函数。

---

**② 为什么需要它？**

**解决什么痛点？**

没有抽象类时，父类的方法体可能是空的或不合理的：

```java
class Animal {
    void sound() {
        // 空实现？动物叫声怎么可能统一？
        // 不写？那子类容易忘记重写
    }
}
```

有了抽象类，明确告诉子类：**这个方法你必须自己实现**。

---

**③ 精简代码示例**

```java
// 抽象类：使用 abstract 关键字
abstract class Animal {
    String name;
    
    // 普通方法：子类可以直接用
    void eat() {
        System.out.println(name + "正在吃东西");
    }
    
    // 抽象方法：没有方法体，子类必须重写
    abstract void sound();
}

// 子类继承抽象类，必须实现所有抽象方法
class Dog extends Animal {
    Dog(String name) {
        this.name = name;
    }
    
    @Override
    void sound() {    // 必须实现
        System.out.println(name + "汪汪叫");
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        // Animal a = new Animal();  // ❌ 错误！抽象类不能实例化
        Animal a = new Dog("旺财");   // ✅ 多态：父类引用指向子类对象
        a.sound();   // 输出：旺财汪汪叫
        a.eat();     // 输出：旺财正在吃东西
    }
}
```

### ② 接口

**① 核心概念**

**一句话总结：**
> 接口是完全抽象的"行为契约"，只声明方法不实现，描述"能做什么"而不关心"怎么做"。

**生活类比：**
- **接口** = USB协议——规定了数据传输的标准，但U盘、鼠标、键盘都可以实现它，各自工作方式不同

**与抽象类的区别：**
- 抽象类：描述"是什么"（is-a关系）→ 狗是一种动物
- 接口：描述"能做什么"（can-do关系）→ 狗能跑、能叫

---

**② 为什么需要它？**

**解决什么痛点？**

Java是单继承，一个类只能继承一个父类。但通过接口，一个类可以实现**多个接口**，弥补了单继承的局限。

```java
// 类只能继承一个父类
class Dog extends Animal { }

// 但可以实现多个接口
class Dog extends Animal implements Runnable, Swimmable { }
```

---

**③ 精简代码示例**

```java
// 定义接口：使用 interface 关键字
interface Runnable {
    void run();    // 接口中的方法自动是 public abstract
}

interface Swimmable {
    void swim();
}

// 类实现接口：使用 implements 关键字
class Dog implements Runnable, Swimmable {
    String name;
    
    Dog(String name) {
        this.name = name;
    }
    
    @Override
    public void run() {
        System.out.println(name + "在奔跑");
    }
    
    @Override
    public void swim() {
        System.out.println(name + "在游泳");
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Dog d = new Dog("旺财");
        d.run();   // 输出：旺财在奔跑
        d.swim();  // 输出：旺财在游泳
        
        // 接口也可以做多态
        Runnable r = new Dog("小黑");
        r.run();   // ✅ 可以调用
        // r.swim(); // ❌ 不能调用，r是Runnable类型
    }
}
```

### ③ 抽象类 vs 接口（重点对比）

| 对比项 | 抽象类 | 接口 |
|--------|--------|------|
| 关键字 | `abstract class` | `interface` |
| 继承/实现 | `extends` | `implements` |
| 多继承 | 单继承 | 多实现 |
| 构造方法 | 有 | 无 |
| 实例变量 | 可以有 | 只能有 `static final` 常量 |
| 普通方法 | 可以有具体实现 | Java 8后可以有 `default`/`static` 方法 |
| 访问修饰符 | 各种都可以 | 默认 `public` |

**选择原则：**
- **用抽象类**：子类有公共代码、共享状态（成员变量）时
- **用接口**：定义能力规范、需要多实现、完全不关心实现细节时

### ④ Java 8 接口新特性：默认方法

**一句话总结：**
> 接口可以有默认实现的方法，避免了修改接口导致所有实现类报错的问题。

```java
interface Vehicle {
    void start();
    
    // 默认方法：提供通用实现，子类可以重写也可以不重写
    default void stop() {
        System.out.println("车辆已停止");
    }
}

class Car implements Vehicle {
    @Override
    public void start() {
        System.out.println("汽车启动");
    }
    // stop() 可以不重写，直接用默认的
}

// 使用
Car c = new Car();
c.start();  // 汽车启动
c.stop();   // 车辆已停止（来自接口默认方法）
```

### ⑤ 小思考题

```java
interface A {
    void method();
}

interface B {
    void method();
}

class C implements A, B {
    // C只需要实现一个method()，同时满足A和B
    @Override
    public void method() {
        System.out.println("实现");
    }
}
```

**思考题：**
```java
// 下面代码能编译通过吗？为什么？
interface Flyable {
    void fly();
}

abstract class Bird {
    abstract void fly();  // 注意：不是 public
}

class Eagle extends Bird implements Flyable {
    // 需要实现几个fly方法？
}
```

提示：注意两个fly方法的访问修饰符差异。

### ⑥ 网络工程场景预告

接口在网络编程中非常常见：

```java
// 定义协议处理器接口
interface ProtocolHandler {
    void handle(byte[] data);
    String getProtocolType();
}

// 不同协议各自实现
class HttpHandler implements ProtocolHandler { ... }
class FtpHandler implements ProtocolHandler { ... }

// 服务器核心：只需要面向接口编程
class Server {
    void registerHandler(ProtocolHandler h) {
        // 不管什么协议，统一处理
    }
}
```
