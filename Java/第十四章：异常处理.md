## 第十四章：异常处理

### ① 核心概念

**一句话总结：**
> 异常是程序运行时发生的"意外情况"，异常处理让程序在出错时能优雅地恢复，而不是直接崩溃。

**生活类比：**
- **无异常处理** = 开车遇到石头直接撞上去，车毁人亡（程序崩溃）
- **有异常处理** = 开车遇到石头，刹车、绕行、或者报修（捕获异常，妥善处理）

**用C的视角理解：**
C语言用返回值（-1、NULL）表示错误，程序员容易忘记检查。
Java用专门的异常机制，强制程序员处理或声明异常。

```c
// C语言风格：容易忘记检查返回值
FILE* fp = fopen("test.txt", "r");
if (fp == NULL) {  // 程序员可能忘记写这个判断
    // 处理错误
}
```

```java
// Java风格：异常被抛出，不处理编译不通过（部分异常）
try {
    FileReader fr = new FileReader("test.txt");
} catch (FileNotFoundException e) {
    // 必须处理，否则编译错误
}
```

### ② 异常体系

**一句话总结：**
> 所有异常都继承自`Throwable`，分为`Error`（系统错误，不可处理）和`Exception`（程序错误，可处理）。

```
Throwable
├── Error（系统级错误，程序无法处理）
│   ├── OutOfMemoryError（内存耗尽）
│   ├── StackOverflowError（栈溢出）
│   └── ...（程序无能为力，通常不做处理）
│
└── Exception（程序级异常，可以处理）
    ├── 受检异常（checked，编译时检查）
    │   ├── IOException（文件不存在等）
    │   ├── SQLException（数据库错误）
    │   └── ...（必须try-catch或throws）
    │
    └── 非受检异常（unchecked，运行时异常）
        ├── NullPointerException（空指针）
        ├── ArrayIndexOutOfBoundsException（数组越界）
        ├── ArithmeticException（除零错误）
        └── ...（可以不处理，但建议处理）
```

**记忆方法：**
- **受检异常**：编译器强制你处理（文件找不到、网络连不上）
- **非受检异常**：程序员自己犯错导致（空指针、除零）

---

### ③ 为什么需要异常处理？

**解决什么痛点？**

没有异常处理时，代码被错误处理占满：

```java
// 没有异常：每个方法都要检查返回值
int result = divide(10, 0);
if (result == -1) {
    System.out.println("除零错误");
} else {
    // 继续处理
}

// 有异常：正常逻辑和错误逻辑分离
try {
    int result = divide(10, 0);
    // 正常逻辑（只有没异常才执行）
} catch (ArithmeticException e) {
    // 错误逻辑（统一处理）
    System.out.println("除零错误：" + e.getMessage());
}
```

**核心作用：**
1. **正常代码和错误代码分离**
2. **错误向上传递**（不用每层都处理）
3. **保证资源释放**（finally块）

---

### ④ 精简代码示例

```java
public class Main {
    public static void main(String[] args) {
        // 场景1：处理除零异常
        try {
            int result = 10 / 0;  // 会抛出 ArithmeticException
            System.out.println("结果：" + result); // 这行不会执行
        } catch (ArithmeticException e) {
            System.out.println("捕获到异常：" + e.getMessage());
        } finally {
            System.out.println("finally块始终执行");
        }
        
        // 场景2：处理数组越界
        try {
            int[] arr = new int[3];
            arr[5] = 10;  // 抛出 ArrayIndexOutOfBoundsException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("数组越界：" + e.getMessage());
        }
        
        // 场景3：多个catch块
        try {
            String str = null;
            str.length();  // 抛出 NullPointerException
        } catch (NullPointerException e) {
            System.out.println("空指针异常");
        } catch (Exception e) {
            System.out.println("其他异常");
        }
        
        System.out.println("程序继续执行");
    }
}
```

### ⑤ try-catch-finally 详解

**语法结构：**
```java
try {
    // 可能出错的代码
} catch (异常类型1 变量名) {
    // 处理异常类型1
} catch (异常类型2 变量名) {
    // 处理异常类型2
} finally {
    // 无论是否异常，都会执行
    // 通常用来释放资源（关闭文件、数据库连接）
}
```

**重要规则：**
```java
// 1. catch顺序：子类异常在前，父类在后
try {
    // ...
} catch (NullPointerException e) {  // 子类在前
    // ...
} catch (Exception e) {              // 父类在后
    // ...
}

// 2. finally一定会执行（除非JVM退出）
try {
    return 1;
} finally {
    return 2;  // 返回值会是2，但不推荐这样写
}

// 3. try-with-resources（自动释放资源，Java7+）
try (FileReader fr = new FileReader("test.txt")) {
    // 使用完自动关闭，不需要finally
} catch (IOException e) {
    e.printStackTrace();
}
```

### ⑥ throws vs throw

| 关键字 | 位置 | 作用 |
|--------|------|------|
| `throws` | 方法声明后 | 声明方法可能抛出异常，交给调用者处理 |
| `throw` | 方法体内 | 手动抛出异常对象 |

```java
class Calculator {
    // 声明抛出异常，交给调用者处理
    public int divide(int a, int b) throws ArithmeticException {
        if (b == 0) {
            // 手动抛出异常
            throw new ArithmeticException("除数不能为0");
        }
        return a / b;
    }
}

// 调用者必须处理
public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        try {
            calc.divide(10, 0);
        } catch (ArithmeticException e) {
            System.out.println("调用者处理：" + e.getMessage());
        }
    }
}
```

### ⑦ 自定义异常

```java
// 1. 继承 Exception（受检异常）或 RuntimeException（非受检）
class AgeOutOfBoundsException extends Exception {
    public AgeOutOfBoundsException(String message) {
        super(message);  // 调用父类构造方法
    }
}

class Person {
    private int age;
    
    public void setAge(int age) throws AgeOutOfBoundsException {
        if (age < 0 || age > 150) {
            throw new AgeOutOfBoundsException("年龄必须在0-150之间：" + age);
        }
        this.age = age;
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Person p = new Person();
        try {
            p.setAge(-5);
        } catch (AgeOutOfBoundsException e) {
            System.out.println(e.getMessage());  // 年龄必须在0-150之间：-5
        }
    }
}
```

### ⑧ 常见异常

| 异常 | 触发场景 |
|------|----------|
| `NullPointerException` | 调用null对象的方法或属性 |
| `ArrayIndexOutOfBoundsException` | 数组索引越界 |
| `ArithmeticException` | 整数除零 |
| `ClassCastException` | 错误的向下转型 |
| `NumberFormatException` | 字符串转数字失败 |
| `IllegalArgumentException` | 传递给方法的参数不合法 |
