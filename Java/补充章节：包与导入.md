## 补充章节：包、导入与protected访问控制

### ① 包（package）

#### 核心概念

**一句话总结：**
> 包是Java中组织类和接口的"文件夹"，用于解决命名冲突、控制访问权限、按功能模块划分代码。

**生活类比：**
- **无包** = 把所有文件扔在一个大抽屉（容易乱，同名文件冲突）
- **有包** = 分门别类的文件夹（`公司/财务/发票.java`、`公司/人事/员工.java`）

**用C的视角理解：**
相当于C语言的头文件路径组织，但Java的包与目录结构严格对应，且有访问控制功能。

---

#### 为什么需要包？

**解决什么痛点？**

```java
// 问题1：类名冲突
// 你写了User类，同事也写了User类，怎么办？
class User { }  // 谁的？

// 解决：用包区分
package com.yourcompany.model;
class User { }

package com.colleagues.model;
class User { }
```

```java
// 问题2：按功能组织代码
// 网络工程专业的项目，可能有多个模块
// model/   - 数据模型
// network/ - 网络通信
// ui/      - 用户界面
// util/    - 工具类
```

**核心作用：**
1. **避免命名冲突**：不同包下可以有同名类
2. **控制访问权限**：`protected`和默认访问级别与包相关
3. **组织代码结构**：按功能模块划分，便于维护

---

#### 包的定义与命名规范

```java
// 包声明必须在文件最顶部（注释之前）
package com.yourcompany.projectname.modulename;

// 命名规范：全部小写，域名倒序
// 例如：com.baidu.mapapi   com.alibaba.druid

// 完整示例
package com.network.chat.client;

public class ChatClient {
    // ...
}
```

**目录结构对应：**
```
src/
└── com/
    └── network/
        └── chat/
            └── client/
                └── ChatClient.java
```

### ② 导入（import）

#### 核心概念

**一句话总结：**
> import用于引入其他包中的类，避免每次使用都写完整包名。

---

#### 为什么需要import？

**解决什么痛点？**

没有import时，使用其他包的类很繁琐：

```java
// 不使用import：每次都写完整包名
public class Main {
    public static void main(String[] args) {
        java.util.ArrayList<String> list = new java.util.ArrayList<>();
        java.util.Date date = new java.util.Date();
        java.io.File file = new java.io.File("test.txt");
        // 代码冗长，难以阅读
    }
}
```

有import后：

```java
// 使用import：简洁清晰
import java.util.ArrayList;
import java.util.Date;
import java.io.File;

public class Main {
    public static void main(String[] args) {
        ArrayList<String> list = new ArrayList<>();
        Date date = new Date();
        File file = new File("test.txt");
    }
}
```

---

#### import语法

```java
// 1. 导入单个类
import java.util.ArrayList;

// 2. 导入包中所有类（不推荐，但方便）
import java.util.*;      // 只导入util包下的类，不递归导入子包

// 3. 导入静态成员（Java 5+）
import static java.lang.Math.PI;
import static java.lang.Math.sqrt;

// 4. 不需要导入的包
// java.lang包（String, System, Thread等）自动导入，无需import

// 5. 同名类冲突时使用完整包名
import java.util.Date;
// import java.sql.Date;  // 冲突！
// 解决：一个用import，另一个用完整包名
java.sql.Date sqlDate = new java.sql.Date(System.currentTimeMillis());
```

---

#### import与package的关系

```java
// 文件：com/network/chat/client/ChatClient.java
package com.network.chat.client;  // 自己所属的包

import com.network.chat.common.Message;  // 引入其他包的类
import java.util.*;                        // 引入Java标准库

public class ChatClient {
    // 可以使用：
    // - 同包下的类（com.network.chat.client.*）不需要import
    // - java.lang包下的类不需要import
    // - 其他包下的类必须import或写完整包名
}
```

### ③ protected访问控制（完整版）

#### 核心概念

**一句话总结：**
> protected修饰的成员可以被**同包类**以及**不同包的子类**访问。

---

#### 回顾四种访问修饰符

| 修饰符 | 同类内 | 同包内 | 不同包子类 | 任何地方 |
|--------|--------|--------|------------|----------|
| `private` | ✅ | ❌ | ❌ | ❌ |
| 默认(无) | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ | ✅ |

**重点理解protected：**
> 比默认多了一个权限——**不同包的子类可以访问父类的protected成员**

---

#### 精简代码示例

```java
// 文件：com/network/animal/Animal.java
package com.network.animal;

public class Animal {
    private int id = 1;           // 只有自己能用
    String name = "动物";          // 默认：同包可用
    protected int age = 3;        // protected：同包 + 不同包子类
    public String type = "生物";   // public：所有人可用
    
    private void privateMethod() { }
    void defaultMethod() { }
    protected void protectedMethod() { }
    public void publicMethod() { }
}
```

```java
// 文件：com/network/animal/Dog.java（同包下）
package com.network.animal;

public class Dog extends Animal {
    public void test() {
        // System.out.println(id);   // ❌ private，不可访问
        System.out.println(name);     // ✅ 默认，同包可访问
        System.out.println(age);      // ✅ protected，同包可访问
        System.out.println(type);     // ✅ public
        
        // defaultMethod();           // ✅ 同包
        // protectedMethod();         // ✅ 同包
        // publicMethod();            // ✅
    }
}
```

```java
// 文件：com/network/zoo/Kennel.java（同包，非子类）
package com.network.zoo;  // 注意：不同包！
import com.network.animal.Animal;

public class Kennel {
    public void test() {
        Animal a = new Animal();
        // System.out.println(a.name);  // ❌ 默认，不同包不可访问
        // System.out.println(a.age);   // ❌ protected，不同包非子类不可访问
        System.out.println(a.type);     // ✅ public
    }
}
```

```java
// 文件：com/network/zoo/Tiger.java（不同包，但是子类）
package com.network.zoo;
import com.network.animal.Animal;

public class Tiger extends Animal {
    public void test() {
        // System.out.println(name);    // ❌ 默认，不同包子类不可访问
        System.out.println(age);        // ✅ protected，不同包子类可访问！
        System.out.println(type);       // ✅ public
        
        // defaultMethod();             // ❌
        protectedMethod();              // ✅ protected，不同包子类可访问
        publicMethod();                // ✅
    }
}
```

#### protected的核心特性

```java
// 关键理解：protected在子类中访问的是继承来的成员，不是通过父类对象访问
package com.example.pkg1;
public class Parent {
    protected String value = "parent";
}

package com.example.pkg2;
import com.example.pkg1.Parent;

public class Child extends Parent {
    public void test() {
        // ✅ 正确：直接访问继承来的protected成员
        System.out.println(value);
        
        // ❌ 错误：不能通过父类对象访问protected成员
        Parent p = new Parent();
        // System.out.println(p.value);  // 编译错误！
    }
}
```
