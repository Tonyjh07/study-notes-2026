## 第六章：类与对象

### ① 核心概念

**一句话总结：**
> 类是对象的“设计图纸”，对象是根据图纸制造的“实体”。

**生活类比：**
- **类** = 手机的设计蓝图（定义了什么功能、什么参数）
- **对象** = 根据蓝图生产出来的一台台真实手机（可以打电话、发短信）

用C的视角理解：类就像是**结构体**，但里面不仅能放数据，还能放函数（方法）。

---

### ② 为什么需要它？

**解决什么痛点？**

C语言中，数据和操作数据的函数是分离的：
```c
// C语言风格
struct Student {
    char name[20];
    int age;
};
void printStudent(struct Student s) { ... }  // 函数在外面
```

Java把数据和操作封装在一起：
```java
// Java风格：学生自己“知道”怎么打印自己
class Student {
    String name;
    int age;
    void print() { ... }  // 行为属于学生本身
}
```

**好处**：代码更符合现实世界的认知，更容易维护和扩展。

---

### ③ 精简代码示例

```java
// 1. 定义类（图纸）
class Student {
    String name;      // 属性（C语言中的成员变量）
    int age;
    
    void study() {    // 方法（C语言中的函数）
        System.out.println(name + "正在学习Java");
    }
}

// 2. 使用类创建对象
public class Main {
    public static void main(String[] args) {
        Student s1 = new Student();  // 创建对象（C语言中叫“实例化”）
        s1.name = "小明";
        s1.age = 20;
        s1.study();                  // 调用方法
    }
}
```

**关键点说明：**
- `new Student()` — 在堆内存中分配空间（相当于C的malloc）
- `Student s1` — 声明一个引用变量（相当于C的指针，但不用写*）
- 访问成员用 `.` 操作符，和C的结构体一样

---


## 第六章：类与对象（续）—— 构造方法

### ① 核心概念

**一句话总结：**
> 构造方法是创建对象时自动执行的“初始化方法”，用来给对象设置初始状态。

**生活类比：**
买车时，4S店交付给你的不只是一堆零件，而是一辆**已经组装好、加满油**的车。构造方法就是那个“组装并初始化”的过程。

**用C的视角理解：**
相当于 `struct Student` 创建后，自动调用一个初始化函数，避免忘记给成员赋值。

---

### ② 为什么需要它？

**解决什么痛点？**

没有构造方法时，创建对象后要手动赋值：
```java
Student s = new Student();
s.name = "小明";    // 容易忘记，导致空指针
s.age = 20;
```

有了构造方法，创建时就能设置：
```java
Student s = new Student("小明", 20);  // 一步到位，不会遗漏
```

**核心作用：**
- 保证对象创建时就处于**有效状态**
- 避免空指针异常（网络编程中常因未初始化引发）

---

### ③ 精简代码示例

```java
class Student {
    String name;
    int age;
    
    // 构造方法：方法名与类名相同，没有返回值类型
    Student(String n, int a) {
        name = n;    // 用参数初始化属性
        age = a;
    }
    
    void study() {
        System.out.println(name + "正在学习");
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Student s = new Student("小明", 20);  // 自动调用构造方法
        s.study();   // 输出：小明正在学习
    }
}
```

**关键点：**
- 如果没写任何构造方法，Java会提供一个**无参构造方法**（默认的）
- 如果写了带参数的构造方法，默认的无参构造就**不再提供**

---

### ④ 小思考题

**思考题：**

## static 关键字

### ① 核心概念

**一句话总结：**
> `static` 表示“属于类本身”而不是“属于某个对象”——类加载时就存在，所有对象共享一份。

**生活类比：**
- **教室里的空调**：所有学生共享一台空调（`static`）
- **学生的书包**：每个学生有自己的书包（非`static`）

用C的视角理解：`static` 成员相当于**全局变量**，但它被组织在类的命名空间里，更安全。

---

### ② 为什么需要它？

**解决什么痛点？**

1. **共享数据**：比如统计创建了多少个对象，需要一个所有对象都能访问的计数器
2. **工具方法**：`Math.sqrt()` 不需要创建Math对象就能直接用
3. **常量定义**：配合 `final` 定义全局常量

---

### ③ 精简代码示例

```java
class Student {
    String name;           // 实例变量：每个对象独有一份
    static int count = 0;  // 静态变量：所有对象共享一份
    
    Student(String n) {
        name = n;
        count++;           // 每次创建对象，计数器+1
    }
    
    void sayHello() {      // 实例方法：需要对象调用
        System.out.println(name + "说你好");
    }
    
    static void showCount() {  // 静态方法：可以直接用类名调用
        System.out.println("学生总数：" + count);
        // 注意：静态方法不能直接访问name（实例变量）
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Student s1 = new Student("小明");
        Student s2 = new Student("小红");
        
        Student.showCount();     // 输出：学生总数：2
        // 等价于：s1.showCount(); 但不推荐这样写
    }
}
```

**内存理解（类比C）：**
- `static` 变量 → 相当于在**静态区/全局区**分配，程序启动时就存在
- 实例变量 → 相当于在**堆区**分配，创建对象时才存在

---

## final 关键字

### ① 核心概念

**一句话总结：**
> `final` 表示“不可改变”——变量变常量、方法禁止重写、类禁止被继承。

**生活类比：**
- **final变量** = 刻在石头上的数字（不能改）
- **final方法** = 家族祖训（子孙不能改）
- **final类** = 密封的盒子（不能再拆开继承）

用C的视角理解：`final` 变量相当于 `const`，但 `final` 类和方法是Java新增的概念。

---

### ② 为什么需要它？

**解决什么痛点？**

1. **定义常量**：`PI = 3.14159` 不应该被修改
2. **安全约束**：某些类不希望被继承（如 `String` 是 `final` 的）
3. **性能优化**：JVM可以对 `final` 方法做内联优化

---

### ③ 精简代码示例

```java
// final变量
class MathConstants {
    final double PI = 3.14159;  // 常量，不能重新赋值
    static final int MAX_SIZE = 1024;  // 常用：全局常量
}

// final类：不能被继承
final class Secret {
    void doSomething() { }
}
// class TryExtend extends Secret { }  // 编译错误！

// final方法：子类不能重写
class Parent {
    final void cannotOverride() { }
}
class Child extends Parent {
    // void cannotOverride() { }  // 编译错误！
}
```

---

## static + final 组合（重点）

### ① 核心概念

**一句话总结：**
> `static final` = 全局常量，属于类且不可变，通常用**全大写+下划线**命名。

---

### ② 为什么需要它？

网络编程中经常定义常量：
- 端口号：`DEFAULT_PORT = 8080`
- 缓冲区大小：`BUFFER_SIZE = 4096`
- 协议标识：`PROTOCOL_HTTP = "HTTP/1.1"`

这些值**整个程序唯一且不变**，用 `static final` 最合适。

---

### ③ 精简代码示例

```java
class NetworkConfig {
    // 命名规范：全大写，下划线分隔
    public static final int DEFAULT_PORT = 8080;
    public static final String SERVER_ADDRESS = "127.0.0.1";
    public static final int MAX_CONNECTIONS = 100;
}

// 使用
public class Main {
    public static void main(String[] args) {
        // 直接用类名访问，不需要创建对象
        System.out.println(NetworkConfig.DEFAULT_PORT);  // 8080
        // NetworkConfig.DEFAULT_PORT = 9090;  // 编译错误！不能修改
    }
}
```

---

## 对比总结

| 关键字 | 修饰变量 | 修饰方法 | 修饰类 |
|--------|----------|----------|--------|
| `static` | 类变量，共享 | 类方法，类名调用 | ❌ 不能修饰类 |
| `final` | 常量，不能改 | 不能被子类重写 | 不能被继承 |
| `static final` | 全局常量 | 类常量方法 | ❌ 不能修饰类 |

## this 关键字

### ① 核心概念

**一句话总结：**
> `this` 代表“当前对象本身”——在方法内部，用来区分成员变量和局部变量，或者调用当前对象的其他方法/构造方法。

**生活类比：**
开会时说“**我**的电脑坏了”——这个“我”就是 `this`，指的是正在说话的那个人。

**用C的视角理解：**
相当于每个成员方法都隐式地传入了一个指向当前结构体的**指针**（`this`），只不过Java不需要你手动写参数。

---

### ② 为什么需要它？

**解决三个痛点：**

1. **参数名和成员变量重名时，需要区分**
2. **在一个构造方法中调用另一个构造方法**（代码复用）
3. **返回当前对象**（实现链式调用，如 `obj.setA().setB()`）

---

### ③ 精简代码示例

```java
class Student {
    String name;
    int age;
    
    // 场景1：区分成员变量和参数
    Student(String name, int age) {
        this.name = name;    // this.name 是成员变量，name 是参数
        this.age = age;
    }
    
    // 场景2：调用另一个构造方法（必须在第一行）
    Student() {
        this("未知", 0);     // 调用上面的有参构造
        System.out.println("无参构造被调用");
    }
    
    // 场景3：返回当前对象，支持链式调用
    Student setName(String name) {
        this.name = name;
        return this;         // 返回当前对象
    }
    
    void print() {
        System.out.println(name + "，" + age + "岁");
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        // 链式调用：连续调用多个方法
        Student s = new Student()
                    .setName("小明")
                    .setName("小红");  // 注意：会覆盖为"小红"
        s.print();  // 输出：小红，0岁
    }
}
```

---

### ④ this 的内存理解

```java
Student s1 = new Student("小明", 20);
Student s2 = new Student("小红", 19);

// 当调用 s1.print() 时：
// print() 方法内部，this 指向 s1 的内存地址
// 所以 this.name 就是 s1.name

// 当调用 s2.print() 时：
// this 指向 s2 的内存地址
```

**用C的视角理解：**
```c
// C语言模拟Java的this
struct Student { char* name; int age; };
void print(struct Student* this) {  // this 就是隐式参数
    printf("%s", this->name);
}
// 调用时：print(&s1);  —— Java的 s1.print() 就是这个意思
```

## 补充：类与对象的完整内存分析

### ① 核心概念

**一句话总结：**
> 引用变量在栈内存，对象实体在堆内存，`static` 成员在方法区。

---

### ② 内存分析示例

```java
class Student {
    String name;          // 实例变量 → 堆内存中每个对象一份
    static int count = 0; // 静态变量 → 方法区，共享
    
    Student(String name) {
        this.name = name;
        count++;
    }
}

// 执行代码
Student s1 = new Student("小明");
Student s2 = new Student("小红");
```

**内存布局：**

```
栈内存                    堆内存                    方法区
┌─────────┐              ┌─────────────────┐     ┌─────────────────┐
│ s1      │──────────────→│ Student对象1    │     │ Student类信息   │
│ (引用)  │              │ name = "小明"   │     │ count = 2       │
├─────────┤              └─────────────────┘     │ (静态变量)      │
│ s2      │──────────────→│ Student对象2    │     └─────────────────┘
│ (引用)  │              │ name = "小红"   │
└─────────┘              └─────────────────┘
```

**关键理解：**
- `s1` 和 `s2` 存的是**地址**（相当于C的指针）
- `new` 在堆内存创建对象，返回地址
- `count` 只有一份，所有对象共享
- 方法调用时，`this` 就是当前对象的地址

---
