## 第八章：继承

### ① 核心概念

**一句话总结：**
> 子类继承父类，自动拥有父类的非私有属性和方法，实现代码复用，并可以扩展或重写父类功能。

**生活类比：**
- **父类** = 手机的基础模型（能打电话、发短信）
- **子类** = 智能手机（继承基础功能，还能上网、拍照）

**用C的视角理解：**
继承就像C语言的结构体嵌套，但Java在语法层面支持更强大的扩展和重写机制。

---

### ② 为什么需要它？

**解决什么痛点？**

没有继承时，重复代码很多：

```java
// 没有继承：代码重复
class Student {
    String name;
    int age;
    void eat() { ... }
    void sleep() { ... }
    void study() { ... }   // 学生特有
}

class Teacher {
    String name;
    int age;
    void eat() { ... }
    void sleep() { ... }
    void teach() { ... }   // 老师特有
}
```

有了继承，公共部分提取到父类：

```java
// 有继承：公共代码只写一次
class Person {
    String name;
    int age;
    void eat() { ... }
    void sleep() { ... }
}

class Student extends Person {
    void study() { ... }    // 自动拥有name、age、eat()、sleep()
}

class Teacher extends Person {
    void teach() { ... }    // 自动拥有name、age、eat()、sleep()
}
```

**核心作用：**
1. **代码复用**：公共部分只写一次
2. **层次结构**：符合现实世界的分类思维
3. **多态基础**：为后续多态做准备

---

### ③ 精简代码示例

```java
// 父类（基类）
class Person {
    String name;
    int age;
    
    void eat() {
        System.out.println(name + "正在吃饭");
    }
}

// 子类（派生类）：extends 表示继承
class Student extends Person {
    String studentId;   // 扩展属性
    
    void study() {      // 扩展方法
        System.out.println(name + "正在学习");
    }
    
    // 重写（Override）父类方法
    @Override
    void eat() {
        System.out.println(name + "在食堂吃饭");
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Student s = new Student();
        s.name = "小明";      // 来自父类
        s.age = 20;          // 来自父类
        s.studentId = "001"; // 自己特有
        
        s.eat();             // 输出：小明在食堂吃饭（重写后的）
        s.study();           // 输出：小明正在学习
    }
}
```

### ④ super 关键字

**① 核心概念**
> `super` 代表父类对象，用于访问父类的成员（属性、方法、构造方法）。

**② 精简代码示例**

```java
class Person {
    String name;
    
    Person(String name) {
        this.name = name;
    }
    
    void show() {
        System.out.println("Person: " + name);
    }
}

class Student extends Person {
    String school;
    
    // 子类构造方法必须调用父类构造方法
    Student(String name, String school) {
        super(name);        // 调用父类构造方法，必须放在第一行
        this.school = school;
    }
    
    @Override
    void show() {
        super.show();       // 调用父类的方法
        System.out.println("Student: " + name + "，" + school);
    }
}
```

**super 的三种用法：**
| 用法 | 示例 | 说明 |
|------|------|------|
| 调用父类构造方法 | `super(参数)` | 必须在子类构造方法的第一行 |
| 调用父类属性 | `super.name` | 当子类属性与父类同名时区分 |
| 调用父类方法 | `super.show()` | 在重写的方法中调用父类原版 |

### ⑤ 方法重写（Override）规则

**一句话总结：**
> 子类重写父类方法时，方法签名必须相同，访问权限不能更严格。

**精简示例：**

```java
class Parent {
    protected void method() { }      // 权限：protected
}

class Child extends Parent {
    @Override
    public void method() { }          // ✅ 可以扩大权限（protected → public）
    
    // @Override
    // private void method() { }      // ❌ 错误！不能缩小权限
}
```

**重写规则：**
- ✅ 方法名、参数列表必须相同
- ✅ 返回值类型相同或是其子类
- ✅ 访问权限不能更严格（可以更宽松）
- ❌ 静态方法不能被重写
- ❌ `final` 方法不能被重写

### ⑥ Object 类

**一句话总结：**
> 所有类都直接或间接继承 `Object`，它是Java的“万类之祖”。

**常用方法：**
```java
class Student {
    String name;
    
    // 重写 toString()：返回对象的字符串表示
    @Override
    public String toString() {
        return "Student: " + name;
    }
    
    // 重写 equals()：比较对象内容是否相等
    @Override
    public boolean equals(Object obj) {
        if (obj instanceof Student) {
            Student s = (Student) obj;
            return this.name.equals(s.name);
        }
        return false;
    }
}

// 使用
Student s1 = new Student();
s1.name = "小明";
Student s2 = new Student();
s2.name = "小明";

System.out.println(s1.toString());  // Student: 小明
System.out.println(s1 == s2);       // false（比较地址）
System.out.println(s1.equals(s2));  // true（比较内容）
```

### ⑦ 小思考题

```java
class Animal {
    String name = "动物";
    void sound() {
        System.out.println("动物叫");
    }
}

class Dog extends Animal {
    String name = "狗";
    void sound() {
        System.out.println("汪汪");
    }
    void bark() {
        System.out.println("小狗吠");
    }
}

Dog d = new Dog();
System.out.println(d.name);      // 输出什么？
d.sound();                       // 输出什么？

Animal a = new Dog();
System.out.println(a.name);      // 输出什么？（注意：属性没有重写）
a.sound();                       // 输出什么？
// a.bark();                     // 这行能编译通过吗？为什么？
```
