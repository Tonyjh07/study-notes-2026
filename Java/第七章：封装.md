## 第七章：封装

### ① 核心概念

**一句话总结：**
> 封装就是把数据（属性）藏起来，只通过规定的方法（getter/setter）来访问，保护数据不被随意修改。

**生活类比：**
- **银行ATM机**：你的账户余额是私有的（private），不能直接改。只能通过“存款”“取款”这些规定方法（public）来操作，ATM会帮你检查余额是否足够。

**用C的视角理解：**
封装就像是C语言里不直接暴露结构体成员，而是提供一组函数来操作它——只是Java用 `private` 和 `public` 关键字在语法层面强制了这种约束。

---

### ② 为什么需要它？

**解决什么痛点？**

没有封装时，数据可以被任意修改，容易出错：

```java
// 没有封装的问题
Student s = new Student();
s.age = -5;      // 直接赋值，年龄可以是负数，不合理！
```

有了封装，可以在setter里做**合法性校验**：

```java
// 有封装的保护
public void setAge(int age) {
    if (age >= 0 && age <= 150) {
        this.age = age;
    } else {
        System.out.println("年龄不合法！");
    }
}
```

**核心作用：**
1. **数据安全**：防止不合理赋值
2. **隐藏实现**：内部逻辑可以随时修改，不影响外部调用
3. **便于维护**：所有访问都经过统一入口

---

### ③ 精简代码示例

```java
class Student {
    // 1. 属性用 private 修饰，外部不能直接访问
    private String name;
    private int age;
    
    // 2. 提供 public 的 getter/setter 方法
    public void setName(String name) {
        this.name = name;
    }
    
    public String getName() {
        return name;
    }
    
    public void setAge(int age) {
        if (age >= 0 && age <= 150) {  // 校验逻辑
            this.age = age;
        } else {
            System.out.println("年龄必须在0-150之间");
        }
    }
    
    public int getAge() {
        return age;
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        Student s = new Student();
        s.setName("小明");
        s.setAge(20);
        
        System.out.println(s.getName() + "，" + s.getAge() + "岁");
        
        s.setAge(-5);  // 输出：年龄必须在0-150之间，数据未被修改
    }
}
```

### ④ 访问修饰符总结

| 修饰符 | 同类内 | 同包内 | 子类 | 任何地方 |
|--------|--------|--------|------|----------|
| `private` | ✅ | ❌ | ❌ | ❌ |
| 默认(不写) | ✅ | ✅ | ❌ | ❌ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| `public` | ✅ | ✅ | ✅ | ✅ |

**简单记忆：**
- `private`：自己用
- `public`：大家用
- 默认：包内用
- `protected`：包内+子孙用（后续继承章节会细讲）

### ⑤ JavaBean规范

在实际开发中（包括你们后续的项目），Java类通常遵循**JavaBean规范**：

```java
// JavaBean标准写法
class Person {
    // 1. 属性私有
    private String name;
    private int age;
    
    // 2. 提供无参构造方法
    public Person() { }
    
    // 3. 提供有参构造方法（可选）
    public Person(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    // 4. 提供getter/setter
    public String getName() { return name; }
    public void setName(String name) { this.name = name; }
    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}
```

**为什么这样写？**
很多框架（如后续可能用到的JSON解析、ORM框架）都依赖这种命名规范来自动处理数据。

### ⑥ 小思考题

```java
class BankAccount {
    private double balance;
    
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }
    
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        }
    }
    
    public double getBalance() {
        return balance;
    }
}
```

**思考题：**
上面代码中，balance是private的。为什么我们不直接提供一个 `setBalance` 方法？如果提供了，会带来什么问题？

提示：想想银行的业务场景——余额应该能被直接赋值吗？
