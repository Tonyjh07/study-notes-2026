## 第十七章：多线程

### ① 核心概念

**一句话总结：**
> 线程是程序中的"轻量级执行单元"，多线程让程序能同时执行多个任务，提高CPU利用率。

**生活类比：**
- **单线程** = 一个人吃三碗面→一碗一碗吃（总时间=三碗时间之和）
- **多线程** = 三个人各吃一碗面→同时吃（总时间=一碗面的时间）

**用C的视角理解：**
Java的线程相当于C语言的`pthread`，但Java内置了更简单的API和跨平台支持。

---

### ② 进程 vs 线程

| 对比 | 进程 | 线程 |
|------|------|------|
| 定义 | 程序的一次执行（重量级） | 进程内的一个执行单元（轻量级） |
| 资源 | 独立的内存空间 | 共享进程的内存空间 |
| 切换开销 | 大 | 小 |
| 通信方式 | IPC（管道、消息队列等） | 直接访问共享变量 |
| Java示例 | 启动一个JVM就是一个进程 | `new Thread().start()` |

**一句话区分：**
> 进程是资源分配的单位，线程是CPU调度的单位。

---

### ③ 为什么需要多线程？

**解决什么痛点？**

单线程做耗时任务时，程序会"卡住"：

```java
// 单线程问题：下载文件时界面卡死
readLargeFile();     // 耗时10秒，界面无响应
updateUI();          // 10秒后才执行
```

多线程让耗时任务在后台执行：

```java
// 多线程方案：后台下载，界面保持响应
new Thread(() -> {
    readLargeFile();   // 后台执行
}).start();
updateUI();            // 立即执行，不等待下载完成
```

**核心作用：**
1. **提高响应速度**：UI不卡顿
2. **充分利用CPU**：多核CPU并行执行
3. **处理并发请求**：Web服务器同时处理多个用户

---

### ④ 创建线程的两种方式

#### 方式1：继承Thread类

```java
class MyThread extends Thread {
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println(Thread.currentThread().getName() + ": " + i);
        }
    }
}

// 使用
MyThread t1 = new MyThread();
MyThread t2 = new MyThread();
t1.start();  // 启动线程（自动调用run方法）
t2.start();
```

#### 方式2：实现Runnable接口（推荐）

```java
class MyRunnable implements Runnable {
    @Override
    public void run() {
        for (int i = 0; i < 5; i++) {
            System.out.println(Thread.currentThread().getName() + ": " + i);
        }
    }
}

// 使用
Thread t1 = new Thread(new MyRunnable());
Thread t2 = new Thread(new MyRunnable());
t1.start();
t2.start();
```

#### Lambda简化版（最简洁）

```java
// 一行创建线程
new Thread(() -> {
    System.out.println("线程执行");
}).start();
```

**两种方式对比：**
| 方式 | 优点 | 缺点 |
|------|------|------|
| 继承Thread | 代码简单 | 单继承限制，不能继承其他类 |
| 实现Runnable | 灵活，可继承其他类 | 代码稍多 |

**推荐：优先使用Runnable**

### ⑤ 线程的生命周期（6种状态）

```
新建(New)
   ↓ start()
就绪(Runnable) ←→ 运行(Running)
   ↓               ↓
等待/阻塞(Waiting/Blocked)
   ↓               ↓
计时等待(Timed Waiting)
   ↓
终止(Terminated)
```

**状态转换：**
| 状态 | 触发条件 |
|------|----------|
| 新建 | `new Thread()` |
| 就绪 | `start()` 调用后 |
| 运行 | CPU调度执行run() |
| 阻塞 | 等待synchronized锁 |
| 等待 | `wait()`/`join()` |
| 计时等待 | `sleep(ms)`/`wait(timeout)` |
| 终止 | run()执行完毕 |

```java
Thread t = new Thread(() -> {
    try {
        Thread.sleep(1000);  // 计时等待
    } catch (InterruptedException e) {
        e.printStackTrace();
    }
});
System.out.println(t.getState());  // NEW
t.start();
System.out.println(t.getState());  // RUNNABLE（可能）
// 1秒后
System.out.println(t.getState());  // TERMINATED
```

### ⑥ 线程常用方法

```java
Thread t = new Thread(() -> {
    // 执行任务
});

t.start();              // 启动线程
t.join();               // 等待t执行完毕（当前线程阻塞）
t.join(1000);           // 最多等待1000毫秒

Thread.sleep(1000);     // 当前线程休眠1秒
Thread.yield();         // 让出CPU时间片（谦让）

t.interrupt();          // 中断线程
t.isInterrupted();      // 检查是否被中断

Thread.currentThread(); // 获取当前线程对象
Thread.currentThread().getName(); // 获取线程名字
```

### ⑦ 线程安全问题（重点）

**问题演示：**

```java
class Counter {
    private int count = 0;
    
    public void increment() {
        count++;  // 非原子操作（读-改-写三步）
    }
    
    public int getCount() {
        return count;
    }
}

// 两个线程同时调用increment 1000次
Counter counter = new Counter();
Thread t1 = new Thread(() -> {
    for (int i = 0; i < 1000; i++) counter.increment();
});
Thread t2 = new Thread(() -> {
    for (int i = 0; i < 1000; i++) counter.increment();
});
t1.start();
t2.start();
t1.join();
t2.join();
System.out.println(counter.getCount());  // 结果不是2000！可能是1993、1987...
```

**为什么会这样？**
`count++` 不是一步完成的：
1. 读取count的值
2. 计算count+1
3. 写回count

两个线程可能同时读到同一个值，导致少加。

---

### ⑧ synchronized 解决方案

**一句话总结：**
> synchronized给代码加锁，同一时刻只有一个线程能执行，保证原子性。

```java
class Counter {
    private int count = 0;
    
    // 方式1：同步方法（锁当前对象）
    public synchronized void increment() {
        count++;
    }
    
    // 方式2：同步代码块（更精细控制）
    public void increment2() {
        synchronized (this) {  // 锁当前对象
            count++;
        }
    }
    
    public synchronized int getCount() {
        return count;
    }
}

// 执行结果：2000 ✅
```

**synchronized原理：**
> 像厕所的门锁——一个人进去后锁门，其他人只能在门外等待，等里面的人出来才能进。

---

### ⑨ volatile 关键字

**一句话总结：**
> volatile保证变量的可见性，但不保证原子性。

```java
class Flag {
    volatile boolean running = true;  // volatile保证多线程可见
}

// 场景：一个线程修改标志，另一个线程看到修改
// 不加volatile，子线程可能永远看不到running=false
```

**synchronized vs volatile：**
| 特性 | synchronized | volatile |
|------|--------------|----------|
| 原子性 | ✅ 保证 | ❌ 不保证 |
| 可见性 | ✅ 保证 | ✅ 保证 |
| 互斥性 | ✅ 线程互斥 | ❌ 不互斥 |
| 性能 | 较慢 | 较快 |

## 第十七章（补充）：多线程进阶

## 一、线程间通信

### ① 核心概念

**一句话总结：**
> 线程间通信是指多个线程通过`wait()`、`notify()`、`notifyAll()`等方法协调执行顺序，避免"忙等待"浪费CPU。

**生活类比：**
- **忙等待** = 你不停问"饭好了吗"（浪费口水/CPU）
- **等待通知** = 你安心睡觉，饭好了妈妈叫你（高效）

---

### ② 为什么需要线程间通信？

**解决什么痛点？**

生产者-消费者问题的经典场景：

```java
// 问题：没有通信，消费者需要不断检查
while (buffer.isEmpty()) {
    // 空循环，浪费CPU
}
// 有数据了才处理
```

有通信后，消费者在没有数据时**主动等待**，生产者生产后**通知**消费者。

---

### ③ 精简代码示例

```java
// 生产者-消费者示例（使用wait/notify）
class DataBuffer {
    private String data;
    private boolean isEmpty = true;
    
    // 生产者放入数据
    public synchronized void produce(String value) throws InterruptedException {
        while (!isEmpty) {      // 数据未消费，等待（用while防止虚假唤醒）
            wait();             // 释放锁，进入等待状态
        }
        data = value;
        isEmpty = false;
        System.out.println("生产: " + value);
        notify();               // 唤醒等待的消费者
    }
    
    // 消费者取出数据
    public synchronized String consume() throws InterruptedException {
        while (isEmpty) {       // 没有数据，等待
            wait();
        }
        String value = data;
        isEmpty = true;
        System.out.println("消费: " + value);
        notify();               // 唤醒等待的生产者
        return value;
    }
}

// 使用
public class Main {
    public static void main(String[] args) {
        DataBuffer buffer = new DataBuffer();
        
        new Thread(() -> {
            for (int i = 1; i <= 5; i++) {
                try { buffer.produce("消息" + i); } 
                catch (InterruptedException e) { }
            }
        }).start();
        
        new Thread(() -> {
            for (int i = 1; i <= 5; i++) {
                try { buffer.consume(); } 
                catch (InterruptedException e) { }
            }
        }).start();
    }
}
```

**wait() 和 notify() 关键点：**
| 方法 | 作用 | 注意事项 |
|------|------|----------|
| `wait()` | 当前线程等待，释放锁 | 必须在synchronized内调用 |
| `notify()` | 随机唤醒一个等待线程 | 必须在synchronized内调用 |
| `notifyAll()` | 唤醒所有等待线程 | 必须在synchronized内调用 |

---

## 二、线程池

### ① 核心概念

**一句话总结：**
> 线程池是预先创建一批线程，重复使用执行任务，避免频繁创建/销毁线程的开销。

**生活类比：**
- **无线程池** = 每次搬砖都临时招人（招人成本高）
- **有线程池** = 养一批固定工人，有活就干（复用，高效）

---

### ② 为什么需要线程池？

**解决什么痛点？**

```java
// 问题：每来一个任务就创建新线程
for (int i = 0; i < 10000; i++) {
    new Thread(() -> doTask()).start();  // 创建10000个线程，内存爆炸
}
```

**线程池的好处：**
1. **降低资源消耗**：复用线程，减少创建销毁开销
2. **提高响应速度**：任务到达时，线程已存在，直接执行
3. **便于管理**：控制最大并发数，防止资源耗尽

---

### ③ 精简代码示例

```java
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;

public class ThreadPoolDemo {
    public static void main(String[] args) {
        // 创建固定大小的线程池（核心）
        ExecutorService pool = Executors.newFixedThreadPool(3);
        
        // 提交10个任务，只有3个线程会轮流执行
        for (int i = 1; i <= 10; i++) {
            final int taskId = i;
            pool.submit(() -> {
                System.out.println(Thread.currentThread().getName() + " 执行任务" + taskId);
                try { Thread.sleep(500); } catch (InterruptedException e) { }
            });
        }
        
        pool.shutdown();  // 关闭线程池（不再接收新任务）
    }
}
```

**常用线程池类型：**
| 方法 | 说明 | 适用场景 |
|------|------|----------|
| `newFixedThreadPool(n)` | 固定大小线程池 | 任务量稳定，控制并发 |
| `newCachedThreadPool()` | 可缓存线程池，需要时创建，空闲回收 | 任务量不确定，短时间任务 |
| `newSingleThreadExecutor()` | 单线程池 | 任务需要顺序执行 |
| `newScheduledThreadPool(n)` | 定时/延迟执行 | 定时任务 |

## 三、线程安全集合

### ① 核心概念

**一句话总结：**
> 普通集合（如ArrayList、HashMap）在多线程环境下不安全，需要使用`ConcurrentHashMap`、`CopyOnWriteArrayList`等线程安全集合。

---

### ② 为什么需要线程安全集合？

```java
// 问题：多线程同时修改ArrayList
List<String> list = new ArrayList<>();
for (int i = 0; i < 1000; i++) {
    new Thread(() -> list.add("A")).start();
}
// 可能报错：ConcurrentModificationException 或 数据丢失
```

---

### ③ 线程安全集合解决方案

```java
import java.util.concurrent.*;
import java.util.*;

public class SafeCollectionDemo {
    public static void main(String[] args) {
        // 1. ConcurrentHashMap（推荐，分段锁/红黑树，性能好）
        ConcurrentHashMap<String, Integer> map = new ConcurrentHashMap<>();
        map.put("key", 1);
        
        // 2. CopyOnWriteArrayList（读多写少场景）
        //   写操作复制整个数组，适合读远多于写的场景
        CopyOnWriteArrayList<String> list = new CopyOnWriteArrayList<>();
        list.add("A");    // 线程安全
        
        // 3. BlockingQueue（阻塞队列，常用于生产者-消费者）
        BlockingQueue<String> queue = new ArrayBlockingQueue<>(10);
        queue.put("任务");      // 队列满时阻塞等待
        String task = queue.take();  // 队列空时阻塞等待
        
        // 4. 传统同步包装（性能较差）
        List<String> syncList = Collections.synchronizedList(new ArrayList<>());
    }
}
```

**线程安全集合对比：**
| 集合 | 普通版 | 线程安全版 | 特点 |
|------|--------|------------|------|
| Map | HashMap | ConcurrentHashMap | 推荐，性能好 |
| List | ArrayList | CopyOnWriteArrayList | 读多写少适用 |
| List | ArrayList | Collections.synchronizedList | 旧方案，性能差 |
| Queue | LinkedList | ArrayBlockingQueue | 阻塞队列 |

## 四、高级锁

### ① ReentrantLock（可重入锁）

**一句话总结：**
> ReentrantLock是synchronized的增强版，支持可中断、超时、公平锁等高级特性。

```java
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

class Counter {
    private int count = 0;
    private Lock lock = new ReentrantLock();
    
    public void increment() {
        lock.lock();  // 加锁
        try {
            count++;
        } finally {
            lock.unlock();  // 必须在finally中释放
        }
    }
    
    // 高级特性：尝试加锁，超时不等待
    public boolean tryIncrement() throws InterruptedException {
        if (lock.tryLock(1, TimeUnit.SECONDS)) {  // 等待1秒
            try {
                count++;
                return true;
            } finally {
                lock.unlock();
            }
        }
        return false;  // 获取锁超时
    }
}
```

### ② ReadWriteLock（读写锁）

**一句话总结：**
> 读锁共享（多线程可同时读），写锁互斥（写时不能读），适合读多写少场景。

```java
import java.util.concurrent.locks.ReentrantReadWriteLock;

class DataCache {
    private Object data;
    private ReentrantReadWriteLock rwLock = new ReentrantReadWriteLock();
    
    public Object read() {
        rwLock.readLock().lock();
        try {
            return data;
        } finally {
            rwLock.readLock().unlock();
        }
    }
    
    public void write(Object newData) {
        rwLock.writeLock().lock();
        try {
            data = newData;
        } finally {
            rwLock.writeLock().unlock();
        }
    }
}
```

### ③ synchronized vs ReentrantLock

| 特性 | synchronized | ReentrantLock |
|------|--------------|---------------|
| 使用方式 | 自动释放锁 | 手动lock/unlock |
| 可中断 | ❌ 不可中断 | ✅ lockInterruptibly() |
| 超时尝试 | ❌ | ✅ tryLock(timeout) |
| 公平锁 | ❌ 非公平 | ✅ 构造参数可指定 |
| 锁绑定条件 | 一个条件 | 多个Condition |
| 性能（早期） | 较差 | 较好 |
| 性能（现在） | 已优化，接近 | 相似 |
| 推荐程度 | 简单场景优先 | 需要高级特性时 |

## 五、死锁及避免方法

### ① 核心概念

**一句话总结：**
> 死锁是两个或多个线程互相持有对方需要的锁，导致所有线程都无法继续执行。

**生活类比：**
> 两个人吃饭，每人只有一根筷子，都等着对方放下筷子给自己——谁也吃不了。

---

### ② 死锁示例

```java
public class DeadlockDemo {
    private static final Object LOCK_A = new Object();
    private static final Object LOCK_B = new Object();
    
    public static void main(String[] args) {
        // 线程1：先拿A锁，再拿B锁
        Thread t1 = new Thread(() -> {
            synchronized (LOCK_A) {
                System.out.println("线程1拿到锁A");
                try { Thread.sleep(100); } catch (InterruptedException e) {}
                synchronized (LOCK_B) {
                    System.out.println("线程1拿到锁B");
                }
            }
        });
        
        // 线程2：先拿B锁，再拿A锁（顺序相反）
        Thread t2 = new Thread(() -> {
            synchronized (LOCK_B) {
                System.out.println("线程2拿到锁B");
                try { Thread.sleep(100); } catch (InterruptedException e) {}
                synchronized (LOCK_A) {
                    System.out.println("线程2拿到锁A");
                }
            }
        });
        
        t1.start();
        t2.start();
        // 可能死锁：t1持有A等B，t2持有B等A
    }
}
```

---

### ③ 死锁产生的四个必要条件

| 条件 | 说明 |
|------|------|
| 互斥 | 资源一次只能被一个线程占用 |
| 持有并等待 | 持有资源的同时等待其他资源 |
| 不可剥夺 | 资源只能由持有者主动释放 |
| 循环等待 | 线程间形成循环等待关系 |

---

### ④ 避免死锁的方法

```java
// 方法1：固定加锁顺序（最常用）
// 所有线程都按相同顺序获取锁（先A后B）
synchronized (LOCK_A) {
    synchronized (LOCK_B) {
        // 安全
    }
}

// 方法2：使用tryLock超时机制
Lock lockA = new ReentrantLock();
Lock lockB = new ReentrantLock();

public void safeMethod() {
    while (true) {
        try {
            if (lockA.tryLock(100, TimeUnit.MILLISECONDS)) {
                try {
                    if (lockB.tryLock(100, TimeUnit.MILLISECONDS)) {
                        try {
                            // 成功获取两把锁，执行业务
                            break;
                        } finally {
                            lockB.unlock();
                        }
                    }
                } finally {
                    lockA.unlock();
                }
            }
        } catch (InterruptedException e) {
            // 重试
        }
    }
}

// 方法3：使用工具类检测
// jstack 命令或 Java Mission Control 检测死锁
```
